いま起きていること ─ “pytest だけが rakuten パッケージを見つけられない”
コマンド	結果	何が違う？
python -m pytest -q	59 passed	Python を直接起動するので -m が自動で CWD (.) を sys.path[0] に追加
pytest -q (スクリプト実行)	ImportError	実行されるのは …\.venv\Scripts\pytest.exe。
この時 sys.path[0] はスクリプトのある …\.venv\Scripts になり、現在の作業ディレクトリは sys.path 末尾にしか入らない or まったく入らない環境がある (windows launcher 仕様)

そのため 外側フォルダ rakuten/ が sys.path に無い ⇒ import rakuten 失敗、となっています。
（python -m pytest は常に OK なので、実装側のパッケージ構成はもう正しく機能しています）

✅ どちらの呼び方でも通るようにする 2 つの実用解
解 1 ― editable install（推奨）
ルートをパッケージとしてインストールし、site-packages から常に読めるようにする。

pyproject.toml を 1 行修正

toml
[tool.poetry]
# …
packages = [{ include = "rakuten" }]   # ← これだけ追加
インストールしなおし

powershell
pip uninstall -y rakuten-monitor 2>$null   # 残骸があれば
pip install -e .
確認

powershell
pytest -q      # → 59 passed
editable install なら pytest／python -m pytest どちらでも安定、
CI や VS Code もインポート補完が効くため長期的に最もクリーンです。
